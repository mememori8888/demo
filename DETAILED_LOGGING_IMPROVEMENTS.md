## 🔍 「どこで止まっているか」を判断する方法

### ケース1: APIトリガーで止まっている場合

```log
[Step 1/3] APIをトリガー中...
  URL数: 500件
  最初のURL: https://maps.google.com/...

❌ HTTP Error 401: Unauthorized
  Response: {"error": "Invalid API token"}
```

**判定:** APIトークンが無効
**対策:** GitHub Secrets の BRIGHTDATA_API_TOKEN を確認

---

### ケース2: スナップショット待機で止まっている場合

```log
[Step 2/3] スナップショット処理待機中...

💭 【確認#1】 処理中: collecting
  経過時間: 0秒

💭 【確認#50】 処理中: collecting
  経過時間: 740秒 (12分20秒)  ← 12分以上待機中

ここから動かない...
```

**判定:** API処理が遅い、またはスナップショート生成がブロックされている
**対策:**
- API_BATCH_SIZE を削減（50 → 25 → 10）
- MAX_WAIT_MINUTES を確認（現在値: 90分）
- BrightData API のステータスを確認

---

### ケース3: データダウンロードで止まっている場合

```log
[Step 3/3] データダウンロード中...

ここから動かない...
```

**判定:** API通信が遅い、またはレスポンスサイズが大きい
**対策:**
- ネットワーク接続を確認
- API_BATCH_SIZE をさらに削減
