# 介護施設データ取得に関する費用お見積もり

**作成日:** 2026年02月06日  
**対象:** 全国の介護施設（居住系）網羅的データ収集プロジェクト

---

## 1. 目的

本レポートは、全国の主要な介護施設（居住系）の情報を網羅的に収集するプロジェクトについて、費用の見積もりをご提示するものです。

---

## 2. 取得戦略

情報の精度と網羅性を最大化するため、以下の3つのカテゴリに分け、それぞれ最適化されたキーワードで検索を実行します。不要な情報（デイサービス、訪問介護など）を的確に除外することで、コスト効率の高いデータ収集を実現します。

### 対象カテゴリと検索キーワード

| カテゴリ | 検索クエリ（実際の設定） |
| :--- | :--- |
| **老人ホーム** | `老人ホーム -不動産 -求人 -紹介センター -デイサービス -訪問 -居宅 -グループホーム -サービス付き高齢者向け住宅` |
| **サービス付き高齢者向け住宅** | `サービス付き高齢者向け住宅 -不動産 -求人 -紹介センター -デイサービス -訪問 -居宅 -グループホーム -老人ホーム` |
| **グループホーム** | `グループホーム -不動産 -求人 -紹介センター -デイサービス -訪問 -居宅 サービス付き高齢者向け住宅 -老人ホーム` |

**除外キーワードの説明:**
- `-不動産`, `-求人`: 施設情報以外のノイズを除外
- `-紹介センター`: 仲介業者を除外
- `-デイサービス`, `-訪問`, `-居宅`: 通所・在宅系サービスを除外
- 各カテゴリ間での重複を防ぐため、他のカテゴリ名も除外

## 3. 費用のお見積もり

### 前提条件

| 項目 | 内容 | 数量 |
| :--- | :--- | :--- |
| **検索キーワード数** | 上記の3カテゴリ | 3 |
| **検索対象地域数** | `settings/address.csv` に基づく市区町村数 | 2,342 |
| **1検索あたりのリクエスト数** | 設定可能（1〜5回を推奨） | 1〜5 |
| **API単価** | BrightData SERP API | $1.5 / 1,000リクエスト |

### 費用試算（シナリオ別）

#### 【シナリオA】最小構成（1リクエスト/地域）

各地域で検索結果の1ページ目（最大20件）のみを取得します。

-   **総リクエスト数**: `2,342 地域` × `3 キーワード` × `1 リクエスト` = **7,026 リクエスト**
-   **合計費用**: `(7,026 / 1,000) × $1.5` = **約 $10.54**

**推奨用途**: まずは全体像を把握したい場合、または予算を最小限に抑えたい場合

---

#### 【シナリオB】標準構成（最大3リクエスト/地域）

各地域で検索結果の最大3ページ目（最大60件）まで取得します。

-   **総リクエスト数（最大）**: `2,342 地域` × `3 キーワード` × `3 リクエスト` = **21,078 リクエスト**
-   **合計費用（最大）**: `(21,078 / 1,000) × $1.5` = **約 $31.62**

**推奨用途**: バランスの取れた選択肢。主要な施設をほぼ網羅できます

---

#### 【シナリオC】最大構成（最大5リクエスト/地域）

各地域で検索結果の最大5ページ目（最大100件）まで取得します。

-   **総リクエスト数（最大）**: `2,342 地域` × `3 キーワード` × `5 リクエスト` = **35,130 リクエスト**
-   **合計費用（最大）**: `(35,130 / 1,000) × $1.5` = **約 $52.70**

**推奨用途**: 最高レベルの網羅性を求める場合。都市部の施設も漏れなく収集できます

---

## 4. 重要：コスト削減機能（自動早期停止）

本システムには、**無駄なリクエストを防ぐインテリジェント機能**が組み込まれています。

### 仕組み

検索中に「これ以上データがない」と判断した場合、設定された最大リクエスト数に達していなくても、**自動的にその地域の検索を停止**します。

### 具体例

**最大5リクエスト設定の場合:**

-   **データが豊富な都市部（例：東京都新宿区）**
    -   1ページ目: 20件 → 2ページ目: 20件 → ... → 5ページ目: 15件
    -   **実際のリクエスト数: 5回** （設定通り）

-   **データが少ない地方（例：過疎地域）**
    -   1ページ目: 5件 → 2ページ目: データなし（自動停止）
    -   **実際のリクエスト数: 2回** （3回分を節約！）

### コスト削減効果

上記の【シナリオC（最大5リクエスト）】で見積もった **$52.70** は、**理論上の最大値**です。

実際には、多くの地域で早期停止が働くため、**実際の費用は $30〜$45 程度**になることが想定されます。

## 5. 結論とご提案

### 推奨プラン

**【シナリオB】標準構成（最大3リクエスト/地域）** をお勧めいたします。

-   **見積もり費用**: 最大 約 $31.62（実際は $20〜$25程度になる見込み）
-   **取得可能データ**: 各地域・各カテゴリで最大60件の施設情報
-   **網羅性**: 主要な施設をほぼカバーでき、都市部でも十分な件数を確保できます

### 柔軟な調整が可能

リクエスト回数は実行時に変更可能です。まずは【シナリオA】で全体像を確認し、必要に応じて【シナリオB】や【シナリオC】にステップアップすることも可能です。

---

## 6. 安全対策

-   **予算上限設定**: BrightData側で予算上限（例: $100）を設定可能。想定外の課金を100%防止します。
-   **リアルタイム監視**: 実行中のリクエスト数をリアルタイムで確認できます。
-   **途中停止可能**: 費用が想定を超えそうな場合、いつでも処理を停止できます。
