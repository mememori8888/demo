# 介護施設取得スクリプト - ローカル実行ガイド

## 概要
Bright Data Discovery APIを使用して介護施設データを取得するスクリプトです。
処理に時間がかかるため、バックグラウンド実行をサポートしています。

## ファイル構成

### メインスクリプト
- `get_care_facilities.py` - 介護施設取得メインスクリプト
- `test_care_facilities.py` - テスト用スクリプト（小規模データで動作確認）

### ローカル実行用スクリプト
- `run_care_facilities_background.sh` - バックグラウンドで実行
- `check_care_facilities_progress.sh` - 進行状況を確認
- `stop_care_facilities.sh` - 実行を停止

## 処理時間の目安

- **APIリクエスト送信:** 約1分
- **データ収集待機:** 2-10分（クエリの範囲による）
- **データダウンロード:** 数秒-1分
- **データ解析・保存:** 数秒

**合計:** 約3-15分/クエリ

複数クエリの場合は順次処理されます。

## 出力データ

### ファイル
- `results/care_facilities.csv` - 取得した介護施設データ

### 列構成（16列）
1. 施設ID
2. 施設名
3. 電話番号
4. 郵便番号
5. 都道府県
6. 市区町村
7. 住所
8. web（ウェブサイト）
9. GoogleMap（URL）
10. ランク（評価）
11. レビュー数
12. カテゴリ
13. 緯度
14. 経度
15. 施設GID
16. 営業ステータス

### ログファイル
- `logs/care_facilities_YYYYMMDD_HHMMSS.log` - 実行ログ（タイムスタンプ付き）

## トラブルシューティング

### エラー: "Customer is not active"
- APIトークンまたはアカウントに問題があります
- Bright Dataダッシュボードでアカウント状態を確認してください

### 処理が長時間終わらない
- Discovery APIの仕様で、2-10分程度かかるのは正常です
- `check_care_facilities_progress.sh` で進行状況を確認してください
- タイムアウト: 30分（スクリプト内で設定）

### プロセスが見つからない
- 既に完了しているか、異常終了しています
- `check_care_facilities_progress.sh` で最終ログを確認してください

## 注意事項

1. **重複チェック**: 既存の `care_facilities.csv` がある場合、GIDで重複を自動除外します
2. **追記モード**: 既存ファイルに新規データを追記します
3. **コスト**: Bright Data APIの利用料金が発生します
4. **データ形式**: 現在、住所などは英語形式で取得されます（`country="JP"` を指定済み）
