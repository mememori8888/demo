## 🌟 システムの概要

**Issue Ops** という仕組みを使って、プログラミング知識がなくても簡単にデータ収集ができます。

### できること

- ✅ 歯科医院やその他業種の施設情報取得
- ✅ ヒートマップデータからの詳細情報取得
- ✅ レビュー情報の収集
- ✅ ヒートマップCSVの生成

### 特徴

- 🎨 **プリセット機能**: よく使う設定を1クリックで適用
- 📧 **メール通知**: 処理開始・完了時に自動でメールが届く
- 📁 **自動保存**: 結果は自動的に保存され、リンクが提示される
- 🔄 **自動更新**: 設定ファイルを指定すれば次回から自動で最新データを使用

## 🔧 事前準備

### 1. GitHub アカウントの準備

- GitHub アカウントが必要です（無料）
- リポジトリへのアクセス権限を管理者から付与してもらってください

### 2. メール通知の設定

- GitHub の通知設定で、Issue のメール通知をオンにしてください
- 設定方法: GitHub → Settings → Notifications → Email

## 🎨 プリセットの使い方

プリセットは、よく使う設定があらかじめ登録されている便利機能です。

### 利用可能なプリセット

WebUI（https://mememori8888.github.io/googlemap/webapp/）にアクセスすると、以下のプリセットが利用できます:

#### 📍 ヒートマップから施設取得

| プリセット名 | 処理内容 | 処理時間 |
|------------|---------|---------|
| 🦷 歯科バッチ実行（2000件） | 2000住所を処理 | 約30分 |
| 🧪 テスト実行（10件） | 10住所を処理して動作確認 | 約5分 |

#### 🏥 施設情報取得

| プリセット名 | 処理内容 | 使用データ |
|------------|---------|-----------|
| 🦷 歯科医院・全国取得 | 全国の住所から歯科医院を検索 | address.csv |

#### ⭐ レビュー取得

| プリセット名 | 処理内容 | 処理件数 |
|------------|---------|---------|
| 🦷 歯科レビュー全件取得 | 歯科医院FIDから全レビュー取得 | 全件 |
| 🧪 テスト実行（100件） | レビュー取得テスト | 100件 |

#### 📊 ヒートマップ生成

| プリセット名 | 処理内容 | 入力ファイル |
|------------|---------|------------|
| 🦷 歯科ヒートマップ生成 | 施設データからヒートマップCSV作成 | dental.csv |

### プリセットの使い方

1. **WebUI にアクセス**: https://mememori8888.github.io/googlemap/webapp/
2. **ワークフローを選択**: 実行したい処理を選ぶ
3. **プリセットを選択**: 「プリセットを選択」から目的のプリセットをクリック
4. **自動入力を確認**: すべてのフィールドが自動で入力されます
5. **Issue を作成**: 「Issue を作成」ボタンをクリック

> 💡 **Tip**: 初めて使う場合は「🧪 テスト実行」プリセットで動作確認することをおすすめします。

## 📝 Issue の作成方法

### 方法1: WebUI を使う（推奨）

1. https://mememori8888.github.io/googlemap/webapp/ にアクセス
2. ワークフローを選択
3. プリセットを選択（または手動で設定）
4. 「Issue を作成」ボタンをクリック
5. GitHub の Issue 作成画面が開きます
6. 内容を確認して「Submit new issue」をクリック


## ✅ 承認と実行

### 実行の流れ

Issue Ops では、安全のため **管理者の承認** が必要です。

#### ステップ1: Issue を作成

1. WebUI または直接 GitHub で Issue を作成
2. Issue 本文に `/run-◯◯` コマンドと設定を含める
3. 「Submit new issue」をクリック

#### ステップ2: 自動プレビュー

Issue 作成後、システムが自動的に以下を投稿します:

```
🔍 ジョブリクエスト受付

ワークフロー: 🗺️ ヒートマップ施設取得

💰 概算コスト見積もり
- 予想リクエスト数: 2000 req
- 予想コスト: $3.00 (レート: $1.50/1k req)
- 内訳: Batch size 2000

ジョブの実行準備が整いました。

---

✅ 管理者の方: 内容を確認後、`/承認` とコメントして実行を承認してください。
❌ キャンセルする場合は、このIssueをクローズしてください。
```

#### ステップ3: 管理者が承認

**管理者（OWNER権限）のみ** が承認できます:

1. Issue の内容とコスト見積もりを確認
2. Issue のコメント欄に `/承認` と入力
3. コメントを投稿

> 💡 **承認権限**: リポジトリのオーナー（`mememori8888`）または OWNER 権限を持つユーザーのみが承認できます。

#### ステップ4: 実行開始

承認コメント後、すぐに実行が開始されます:

```
🚀 🗺️ ヒートマップ施設取得 の実行を開始します。

完了までしばらくお待ちください...

[Actions ログ](リンク)
```

#### ステップ5: 完了通知

処理完了後、結果が Issue に投稿されます:

```
✅ ジョブ完了

結果は Actions から確認できます。
```

Issue は自動的にクローズされます。

### 承認のキャンセル

実行前にキャンセルしたい場合:
- Issue をクローズする（承認前のみ有効）

### 実行状態の確認

#### 📧 メール通知

- **処理開始時**: 「処理を開始しました」というコメントが Issue に投稿され、メールが届きます
- **処理完了時**: 「処理が完了しました」というコメントが Issue に投稿され、メールが届きます

#### 🔍 Issue での確認

Issue を開くと、以下の情報が表示されます:

```
✅ 処理が完了しました

📁 出力ファイル:
- https://github.com/mememori8888/googlemap/blob/main/results/dental.csv

実行時間: 約30分
処理件数: 2000件
```

## 🔄 設定ファイルの自動更新

### 自動更新とは？

**出力ファイルを設定ファイルとして指定すると、次回の実行時に自動的に最新データが使われます。**

### 使用例

#### シナリオ: 施設情報からFIDを取得し、レビューを収集

1. **施設情報を取得**
   - ワークフロー: `facility`
   - 出力ファイル: `results/dental.csv`

2. **FIDを抽出**
   - ワークフロー: `generate_heatmap`
   - 入力ファイル: `results/dental.csv`
   - 出力ファイル: `results/dental_fid.csv` ← **FIDが自動生成される（dental.csvから自動命名）**

3. **レビューを取得**
   - ワークフロー: `reviews`
   - FIDファイル: `results/dental_fid.csv` ← **自動的に最新のFIDが使われる**
   - 出力ファイル: `results/dental_review.csv`

### 設定ファイルの種類

| ファイル | 用途 |
|---------|----- |
| `settings/address.csv` | 住所データ（全国の市区町村） |
| `settings/address_wordpress_only.csv` | WordPress専用住所データ |
| `settings/exclude_gids.csv` | 除外するGIDリスト |
| `results/dental_fid.csv` | 歯科医院のFID一覧（レビュー取得に必要、自動生成） |

### メリット

- ✅ **手動更新不要**: 最新データを自動で使用
- ✅ **ミス防止**: 古いデータを使う心配がない
- ✅ **効率化**: データの連携がスムーズ

## ⚠️ レビュー取得の注意事項

### FID とは？

**FID (Feature ID)** は、Google Maps でレビューを取得するために必要な一意の識別子です。

### 重要なポイント

#### ❌ GID では レビュー取得できません

- Google Maps API で取得したデータには **GID** が含まれています
- しかし、レビュー取得には **FID** が必要です
- GID と FID は **別のもの** です

#### ✅ FID の取得方法

FIDの取得方法について、**詳しくはお問い合わせください。**

レビュー取得ワークフローの実行には、事前にFIDファイル（`dental_fid.csv`など）が必要です。

## ❓ よくある質問

### Q1: プリセットは追加できますか？

**A**: はい、追加可能です。

開発者に以下の情報を伝えてください:

- プリセット名（例: 「横浜市のみ取得」）
- 使用するファイル（例: `settings/yokohama_address.csv`）
- 処理件数（例: 500件）
- その他の設定


### Q2: 処理が失敗した場合は？

**A**: Issue にエラーメッセージが投稿されます。

- エラー内容を確認してください
- 入力ファイルが存在するか確認してください
- 不明な場合は開発者に Issue のリンクを共有してください

### Q3: 処理時間の目安は？

**A**: 以下を参考にしてください:

| 処理件数 | 推定時間 |
|---------|---------|
| 10件 | 約5分 |
| 100件 | 約10分 |
| 2000件 | 約30分 |
| 10000件 | 約5時間 |

※ ネットワーク状況により変動します

### Q4: 複数の Issue を同時に作成できますか？

**A**: はい、可能です。

ただし、GitHub Actions の同時実行数に制限があるため、順番に処理されることがあります。

### Q5: Issue を削除したら処理も止まりますか？

**A**: いいえ、処理は継続されます。

Issue はトリガーとして機能するだけで、削除しても実行中の処理には影響しません。

### Q6: 以前の結果を再利用できますか？

**A**: はい、できます。

`results/` フォルダ内のファイルは保存されているため、過去の出力ファイルを指定して再実行できます。
