# dental_new.csv を6時間以内に処理する方法

## 概要

dental_new.csv（約70,000行）を複数のワークフローで並行処理することで、6時間以内に完了させます。

## 事前準備：行数確認と最適な分割数の計算

```bash
bash check_dental_new_rows.sh
```

このコマンドで以下が表示されます：
- 総行数
- 最適な分割数（自動計算）
- 各ワークフローが処理する範囲
- 推定処理時間
- 具体的な設定方法

## 処理時間の計算

### パラメータ設定
- **待機時間**: 1秒（delay=1）
- **並列実行数**: 8（workers=8）
- **1行あたりの処理時間**: 約1.5秒

### 6時間制限での処理可能行数
- 6時間 = 21,600秒
- 21,600秒 ÷ 1.5秒/行 = 14,400行/ワークフロー
- 余裕を持って **12,000行/ワークフロー** で設計

### 必要な分割数
- 70,000行の場合: 6分割（各約11,667行）
- 推定時間: 約4.9時間/ワークフロー

## 実行手順（70,000行の場合 - 6分割）

- 並列実行数: `8`

複数のブラウザタブを使うと効率的です。全6個が並行して実行され、約5時間で完了します。

## パラメータの説明

### 処理開始行・処理終了行

- ヘッダー行を除く、実際のデータ行の番号
- 1から開始（CSVの2行目がデータの1行目）
- 例: 総行数70,000行を6分割の場合
  - ワークフロー1: 開始行=1, 終了行=11667
  - ワークフロー2: 開始行=11668, 終了行=23334
  - ワークフロー3: 開始行=23335, 終了行=35001
  - ワークフロー4: 開始行=35002, 終了行=46668
  - ワークフロー5: 開始行=46669, 終了行=58335
  - ワークフロー6: 開始行=58336, 終了行=70000

### バックアップを作成しない

- ワークフロー1: チェックを**外す**（バックアップ作成）
- ワークフロー2〜6: チェックを**入れる**（バックアップ不要、衝突回避）

### 待機時間と並列実行数

- **待機時間: 1秒** - リクエスト間隔を短縮して高速化
- **並列実行数: 8** - 同時処理数を増やして高速化
- ⚠️ これ以上増やすとGoogleからブロックされる可能性があります

## 処理時間の目安

| 総行数 | 分割数 | 1ワークフローあたり | 並列実行時の合計時間 |
|-------|-------|------------------|---------------------|
| 10,000行 | 1分割 | 約4.2時間 | 約4.2時間 |
| 30,000行 | 3分割 | 約4.2時間 | 約4.2時間 |
| 50,000行 | 5分割 | 約4.2時間 | 約4.2時間 |
| 70,000行 | 6分割 | 約4.9時間 | 約4.9時間 |
| 84,000行 | 7分割 | 約5.0時間 | 約5.0時間 |

※ workers=8, delay=1の場合、1行あたり約1.5秒で計算

## 処理完了後

両方のワークフローが完了したら：

1. 「Artifacts」セクションから結果をダウンロード
2. dental_new.csvが更新されています
3. バックアップも保存されています

## トラブルシューティング

### ファイル競合が発生した場合

両方のワークフローが同じファイルに書き込むため、まれに競合が発生する可能性があります。

**対策**:
- 一方のワークフローは `--no-backup` を使用
- 両方のワークフローをできるだけ同時に開始
- GitHub Actionsのpush時にリトライ機能が自動的に働きます

### 処理が6時間を超える場合

70,000行を超える場合は、さらに分割が必要です：

- **80,000行**: 7分割（各11,429行、約5.0時間/ワークフロー）
- **90,000行**: 8分割（各11,250行、約4.7時間/ワークフロー）
- **100,000行**: 9分割（各11,111行、約4.6時間/ワークフロー）

`check_dental_new_rows.sh` を実行すると、自動的に最適な分割数が計算されます。

## コマンドライン実行（ローカルでテスト）

```bash
# 1つ目の範囲（1-11667行）
python update_googlemap_urls.py \
  --input results/dental_new.csv \
  --start-row 1 \
  --end-row 11667 \
  --delay 1 \
  --workers 8

# 2つ目の範囲（11668-23334行）
python update_googlemap_urls.py \
  --input results/dental_new.csv \
  --start-row 11668 \
  --end-row 23334 \
  --delay 1 \
  --workers 8 \
  --no-backup

# ... 残りも同様に実行
```
