# 動的リクエスト最適化機能

## 概要

`address.csv`に基づいて施設検索を行う際、既存の施設データ量を分析し、各エリアのリクエスト回数を自動的に調整する機能です。

---

## 仕組み

### 1. 既存データの分析

プログラム起動時に`facility_file`（例：`dental.csv`）を読み込み、各エリア（都道府県＋市区町村）ごとの施設数を集計します。

```
例：
- 東京都 新宿区: 45件
- 北海道 札幌市: 619件
- 三重県 鈴鹿市: 12件
```

### 2. リクエスト回数の動的決定

`address.csv`の各行を処理する際、既存施設数に応じてリクエスト回数を決定します。

**決定ロジック：**

| 既存施設数 | リクエスト回数 | 取得見込み件数 | 用途 |
|----------|------------|------------|------|
| データなし | 0回（スキップ） | - | 該当エリアに施設が存在しない |
| 0件 | 0回（スキップ） | - | 以前の検索で0件確認済み |
| 1-20件 | 1回 | 最大20件 | 小規模エリア |
| 21-40件 | 2回 | 最大40件 | 中規模エリア |
| 41-60件 | 3回 | 最大60件 | やや大規模エリア |
| 61-80件 | 4回 | 最大80件 | 大規模エリア |
| 81件以上 | 5回 | 最大100件 | 非常に大規模エリア |

※1回のリクエストで最大20件の施設データを取得

### 3. 実行処理

- **スキップ**: データなし/0件のエリアはAPI呼び出しを行わない
- **リクエスト実行**: 1-5回のリクエストでデータを更新・補完

---

## 使用方法

### プログラム

**`facility_BrightData_20_update.py`**

通常の`address.csv`を使用しながら、動的にリクエスト回数を調整します。

### 実行コマンド

```bash
python facility_BrightData_20_update.py
```

### settings.json設定

```json
{
  "task_name": "dental_search",
  "address_csv_path": "address.csv",
  "facility_file": "dental.csv",
  ...
}
```

※`address_csv_path`には通常の住所リストを指定

## コスト削減効果

### 例：歯科医院データ（17,973施設、既存513エリア）

**従来の方法**（全address.csv一律1回リクエスト）:
- address.csv約2,300行 × 1回 = **約2,300回のリクエスト**

**動的リクエスト最適化**:
- address.csv約2,300行を処理
- `facility_file`にデータがないエリア: スキップ（約1,787エリア、0回）
- `facility_file`にデータがあるエリア（513エリア）:
  - 1-20件: 1回 × 約270エリア = 270回
  - 21-40件: 2回 × 約150エリア = 300回
  - 41-60件: 3回 × 約40エリア = 120回
  - 61-80件: 4回 × 約16エリア = 64回
  - 81件以上: 5回 × 約7エリア = 35回
- **合計: 約789回のリクエスト**

**削減効果**: 約2,300回 → 約789回 = **約65%削減**

## 技術詳細

### 実装ロジック

```python
# 1. facility_fileから施設数を集計
facility_counts = {}
with open('results/dental.csv') as f:
    for row in csv.DictReader(f):
        pref = row['都道府県']
        city = row['市区町村']
        key = f"{pref} {city}"
        facility_counts[key] = facility_counts.get(key, 0) + 1

# 2. address.csvの各エリアを処理
for address in address_list:
    estimated_count = facility_counts.get(address.strip(), None)
    
    # リクエスト回数を決定
    if estimated_count is None or estimated_count == 0:
        continue  # スキップ
    elif estimated_count <= 20:
        request_count = 1
    elif estimated_count <= 40:
        request_count = 2
    # ... 以下続く
```

## 利用シーン

### ✅ 適している場合

- **定期的なデータ更新**: 既存データの更新・補完
- **増分データ収集**: 新規施設や情報更新の取得
- **コスト削減重視**: 不要なリクエストを避けたい

### ❌ 適していない場合

- **初回データ収集**: 施設データが存在しない場合
  - → `facility_BrightData_20.py`（固定1回リクエスト版）を使用

---

## メリット

✅ **コスト最適化**: エリアごとに必要最小限のリクエスト  
✅ **自動判定**: 既存データに基づき自動的にリクエスト回数を調整  
✅ **無駄削減**: 施設がないエリアはスキップ  
✅ **柔軟性**: 大規模エリアには最大5回までリクエスト可能  

---

## 注意点

⚠️ **初回実行時**: `facility_file`が存在しない場合、全エリアがスキップされます。まず`facility_BrightData_20.py`で初回データ収集を行ってください。

⚠️ **データなしのスキップ**: `facility_file`に該当エリアのデータがない場合、そのエリアはスキップされます。新しいエリアを追加する場合は固定版を使用してください。
